# 64-tap 16-bit FIR Digital Filter Core

A 64-tap FIR filter RTL project with dual-clock data path:
- `clk1` domain: input data streaming
- `clk2` domain: coefficient preload + MAC computation

The design includes RTL, testbench, synthesis/STA scripts, and example input/output data files.

## Features

- 16-bit signed input/output (`Q1.15` style bit pattern handling in TB)
- 64-tap coefficient memory (`CMEM`)
- CDC FIFO between `clk1` and `clk2`
- FSM-controlled sequence: preload -> wait data -> MAC -> shift/output
- Testbench supports:
  - loading coefficients from `CMEM.txt`
  - loading input stream from `DATA.txt`
  - dumping outputs to `fir_out.txt`
  - optional waveform dump `tb_fir_top.vcd`

## Top-Level Interface (`FIR_top`)

- Inputs
  - `rst_n`: active-low reset
  - `clk1`: input stream clock
  - `clk2`: compute/control clock
  - `valid_in`: input sample valid (`clk1` domain)
  - `din[15:0]`: input sample
  - `cload`: coefficient preload enable (`clk2` domain)
  - `caddr[5:0]`: coefficient address (0~63)
  - `cin[15:0]`: coefficient data
- Outputs
  - `dout[15:0]`: FIR output sample
  - `valid_out`: output valid pulse
  - `done`: end flag from FSM

## Data File Format

- `CMEM.txt`: 64 lines, each line is one 16-bit binary coefficient
- `DATA.txt`: 10000 lines, each line is one 16-bit binary input sample
- `fir_out.txt`: output samples in 16-bit binary format (generated by TB)

## Simulation

Run in `FIR/` directory.

### Option A: Icarus Verilog (iverilog + vvp)

```bash
cd FIR
iverilog -g2012 -o simv \
  RTL/FIR_top.v RTL/FSM.v RTL/ALU.v RTL/CDC_FIFO.v RTL/CMEM.v RTL/shift_reg.v \
  tb_FIR_top.v
vvp simv
```

Outputs:
- `fir_out.txt` (captured valid outputs)
- `tb_fir_top.vcd` (waveform dump if enabled in TB)

### Option B: ModelSim/Questa (example)

```tcl
vlog RTL/*.v tb_FIR_top.v
vsim tb_FIR_top
run -all
```

## Synthesis and STA Scripts (Synopsys)

- `FIR_top(dc).tcl`: Design Compiler script
- `FIR_top(pt).tcl`: PrimeTime STA/power script
- `timing.tcl`: timing constraints (two asynchronous clocks)

Notes:
- These scripts reference course/lab library paths and legacy folder structure.
- You may need to update `search_path`, library/db names, and RTL netlist paths for your local environment.
